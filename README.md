# Симулятор курьера

## Описание проекта

**Симулятор курьера** — это 2D-игра на **C# + Windows Forms**, в которой игрок управляет курьером и развозит заказы по адресам в течение смены.

Главная особенность игры заключается в том, что разные заказы можно доставить только в определённое время суток. Поэтому игроку нужно не просто ездить по карте, а учитывать:
- текущее время суток
- доступность клиентов
- ограничение по времени всей смены

Цель игры — доставить все заказы до конца смены.

---

## Жанр

Мини-игра / симулятор курьера / аркада с элементами тайм-менеджмента

---

## Основные механики

В игре реализованы следующие механики:

- перемещение курьера по карте
- карта с домами и дорогами
- смена времени суток:
  - утро
  - день
  - вечер
  - ночь
- список заказов
- доставка заказов по кнопке `E`
- ограничения по времени доставки
- победа при выполнении всех заказов
- поражение, если время смены закончилось
- рестарт игры по кнопке `R`

---

## Управление

- `W`, `A`, `S`, `D` или стрелки — передвижение
- `E` — передать заказ
- `R` — начать новую смену
- `Enter` — начать игру со стартового экрана

---

## Цель игрока

Игрок должен доставить все заказы до окончания смены.

Сложность заключается в том, что не каждый клиент доступен в любое время.  
Например:
- один заказ можно доставить только утром
- другой — только днём
- третий — вечером
- некоторые доступны в нескольких временных фазах

Поэтому игроку нужно планировать маршрут и учитывать время суток.

## Структура проекта

Проект состоит из двух частей:

### 1. Основной игровой проект `TimePostman`
Содержит:
- форму игры
- отображение карты
- передвижение курьера
- пользовательский интерфейс
- игровую логику

Основные файлы:
- `Form1.cs` — основная форма игры
- `GameSession.cs` — логика игровой сессии
- `DeliveryOrder.cs` — модель заказа
- `DeliveryAttemptResult.cs` — результат попытки доставки
- `TimePhase.cs` — фазы времени суток
- `Program.cs` — точка входа

### 2. Тестовый проект `TestProject.Tests`
Содержит автоматические тесты для проверки игровой логики.

Основной файл:
- `GameSessionTests.cs`

---

## Архитектура

Во время разработки логика игры была вынесена из формы в отдельный класс `GameSession`.

Это позволило:
- отделить интерфейс от игровой логики
- упростить поддержку проекта
- добавить автоматические тесты
- сделать код более понятным

Форма отвечает в основном за:
- отрисовку карты
- ввод с клавиатуры
- отображение информации на экране

Класс `GameSession` отвечает за:
- текущее время суток
- таймеры
- список заказов
- правила доставки
- победу и поражение
- сброс состояния игры

---

## Тестирование

Для проекта был создан отдельный тестовый проект с кодовыми тестами.

Тесты проверяют:
- старт игры с правильного времени суток
- корректную смену фаз времени
- доставку заказа в правильное время
- отказ в доставке в неправильное время
- невозможность повторной доставки одного и того же заказа
- победу при выполнении всех заказов
- поражение при окончании времени
- корректный сброс состояния игры
