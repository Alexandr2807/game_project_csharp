# Симулятор курьера

## О проекте

**Симулятор курьера** — это 2D-игра на **C# + Windows Forms**, в которой игрок управляет курьером и закрывает заказы по адресам в течение одной смены.

Главная особенность игры — зависимость доставки от времени суток. Не каждый клиент доступен всегда, поэтому игроку нужно следить за:
- текущей фазой времени
- доступностью клиента
- общим таймером смены
- маршрутом между домами

Цель игры — доставить все заказы до конца смены.

---

## Жанр

Мини-игра / симулятор курьера / тайм-менеджмент

---

## Основные механики

В игре реализованы:
- перемещение курьера по карте
- игровая карта с домами, дорогами и окружением
- 4 фазы времени суток:
  - утро
  - день
  - вечер
  - ночь
- список заказов с ограничениями по времени доставки
- передача заказа по кнопке `E`
- победа при выполнении всех заказов
- поражение, если время смены закончилось
- рестарт игры по кнопке `R`
- стартовый экран перед началом смены
- финальный экран с итогом смены

---

## Управление

- `W`, `A`, `S`, `D` или стрелки — движение
- `E` — передать заказ
- `R` — начать новую смену
- `Enter` — начать игру со стартового экрана

## Архитектура проекта

Проект собран по упрощенной схеме **MVC**.

### Model
Отвечает за игровые данные и правила:
- `GameSession.cs` — состояние игровой сессии, время суток, таймеры, список заказов, победа и поражение
- `DeliveryOrder.cs` — модель заказа
- `DeliveryAttemptResult.cs` — результат попытки доставки
- `TimePhase.cs` — фазы времени суток
- `GameMap.cs` — данные карты: дома, дороги, деревья, стартовая позиция

### View
Отвечает за отображение:
- `Form1.cs` — отрисовка карты, интерфейса, стартового и финального экранов

### Controller
Отвечает за управление игрой:
- `GameController.cs` — движение курьера, обработка ввода, взаимодействие с домами, подсказки и статусные сообщения

Такое разделение позволило отделить интерфейс от игровой логики и сделать проект понятнее и удобнее для защиты.

## Тестирование

Для проекта создан отдельный тестовый проект **TimePostman.Tests**.

Тесты проверяют:
- старт игры с правильной фазы времени
- смену времени суток по кругу
- успешную доставку в правильное время
- отказ в доставке в неправильное время
- запрет повторной доставки одного и того же заказа
- победу при выполнении всех заказов
- поражение при окончании времени
- корректный сброс состояния игры

Основной файл тестов:
- `GameSessionTests.cs`
